# Daily Rep UX backlog (research-driven)

This backlog consolidates 37 raw recommendations from 5 evaluators into a single prioritized list. It is grounded in lifting-app forum/Reddit research (Hevy, Strong, Stronglifts, FitNotes, Fitbod) and the recurring churn patterns those communities name: rest timers that go silent when backgrounded, fragile daily streaks that punish planned rest, accidental no-undo deletes, too-many-taps to log a set, missing per-lift progression charts, opaque recommendations, and dead-end empty states. Duplicates across evaluators have been merged (files combined), already-shipped items dropped, and everything ranked by impact-to-effort so the cheapest high-impact wins surface first. Every item names what to build and which file(s) to touch.

Each item is ordered within its section by impact/effort. Effort: S (small), M (medium), L (large).
